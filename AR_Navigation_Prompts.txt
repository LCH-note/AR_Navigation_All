📝 STEP 1: 기본 데이터 구조 및 설정
================================================================================

Unity C# 스크립트를 작성해주세요.

목표: AR 맵 관리 시스템의 기본 데이터 구조 정의

요구사항:
1. MapSettings 클래스: 전체 맵의 물리적 크기 정보
   - 맵 너비, 높이 (실제 미터 단위)
   - Canvas 표시 영역 정보

2. Anchor 클래스: 기준점 정보
   - mapID (어느 맵인지)
   - unifiedPosition (통합 좌표에서의 위치)
   - name (앵커의 이름)

3. ScannedRegion 클래스: 스캔된 영역 정보
   - regionName (영역 이름)
   - topLeft, bottomRight (평면도에서의 바운딩 박스)
   - isScanned (스캔 여부)
   - immersalMapIDs (포함된 Immersal 맵 ID들)

4. DocentPanel 클래스: 도슨트 패널 정보
   - id, mapID
   - relativePosition (기준점 대비 상대 좌표)
   - content (설명 텍스트)

5. NavigationWaypoint 클래스: 네비게이션 포인트
   - id, mapID
   - relativePosition (상대 좌표)

직렬화 가능하고 Inspector에서 편집 가능하게 해주세요.

📝 STEP 2: 통합 좌표계 시스템
================================================================================

Unity C# 스크립트를 작성해주세요.

목표: 통합 좌표계 관리 시스템 구현

요구사항:
1. Anchor 위치 관리
   - 각 맵의 기준점(Anchor)의 통합 좌표 저장
   - Anchor 추가/조회 기능
   - 기본값으로 Anchor1=(0,0,0), Anchor2=(20,0,0) 설정

2. 좌표 변환 함수들:
   - LocalToUnified(int mapID, Vector3 localPos): 로컬 → 통합
   - UnifiedToLocal(int mapID, Vector3 unifiedPos): 통합 → 로컬
   - LocalTo2D(Vector3 unifiedPos): 3D 통합 → 2D (X, Z만 사용)

3. 디버그 기능:
   - 변환 과정을 로그로 출력
   - 현재 앵커 정보 출력
   - 좌표 검증 (범위 체크)

4. 사전 설정된 예시 데이터:
   - 박물관 전시실 상황 기준
   - 맵1 포인트들 (5,0,10), (10,0,15)
   - 맵2 포인트들 (5,0,5), (10,0,10)

클래스명: UnifiedCoordinateSystem

📝 STEP 3: Immersal 좌표 변환
================================================================================

Unity C# 스크립트를 작성해주세요.

목표: Immersal SDK의 위치 정보를 통합 좌표계로 변환

요구사항:
1. ImmersalLocationConverter 클래스
   - UnifiedCoordinateSystem 참조
   - 현재 활성 맵 ID 저장

2. 핵심 함수들:
   - GetImmersalPosition(): Immersal에서 사용자 위치 가져오기 (시뮬레이션)
   - GetActiveMapID(): 현재 활성 맵 ID 반환 (시뮬레이션)
   - ConvertToUnified(): Immersal 위치 → 통합 좌표
   - HasValidPose(): 유효한 위치 정보 여부

3. 시뮬레이션 데이터:
   - 맵1에서 사용자가 (3, 0, 7)에 위치
   - 맵2에서 사용자가 (4, 0, 6)에 위치
   - 시뮬레이션으로 매초 약간씩 변경

4. 에러 처리:
   - Localization 실패 상황 처리
   - 맵 전환 감지

클래스명: ImmersalLocationConverter

📝 STEP 4: 크로스맵 네비게이션
================================================================================

Unity C# 스크립트를 작성해주세요.

목표: 두 개 맵을 연결하는 네비게이션 경로 관리 및 시각화

요구사항:
1. NavigationPathManager 클래스
   - 맵1과 맵2를 연결하는 경로 정의
   - 각 맵의 상대 좌표 저장

2. 경로 데이터:
   - 맵1 경로: [(5,0,5), (10,0,10)]
   - 맵2 경로: [(0,0,0), (5,0,5)]
   
3. 핵심 함수들:
   - GetCrossMapPath(): 전체 경로 반환
   - ConvertPathToUnified(): 경로 전체를 통합 좌표로 변환
   - GetPathSegmentByMap(int mapID): 특정 맵의 경로 구간만
   - CalculateTotalDistance(): 전체 거리 계산

4. 시각화 정보:
   - 각 포인트의 맵 ID 저장
   - 포인트 간 거리 계산
   - 경로의 유효성 검증

5. 출력 예시:
   - "경로 포인트 1: (통합좌표)"
   - "경로 포인트 2: (통합좌표)"
   - "총 거리: XXm"

클래스명: NavigationPathManager

📝 STEP 5: 2D 맵 디스플레이 시스템
================================================================================

Unity C# 스크립트를 작성해주세요.

목표: 3D 통합 좌표를 2D 지도에 변환하여 표시

요구사항:
1. CoordinateDisplay 클래스
   - 3D → 2D 변환 로직
   - 2D → Canvas 픽셀 좌표 변환

2. 변환 함수들:
   - UnifiedTo2D(Vector3): 3D 통합 좌표 → 2D (X,Z만)
   - WorldToCanvasPosition(Vector2): 실제 거리 → Canvas 픽셀
   - CanvasToWorld(Vector2): 역변환 (Canvas → 실제 거리)

3. 설정 값들:
   - 맵 물리적 크기: 50m × 40m
   - Canvas 표시 영역: 800 × 600 픽셀
   - 맵 원점: (0, 0)

4. 예시 데이터로 변환:
   - 포인트 (5, 10) (3D) → (5, 10) (2D) → Canvas 좌표
   - 포인트 (25, 15) (3D) → (25, 15) (2D) → Canvas 좌표

5. 검증 기능:
   - 캔버스 범위 내인지 확인
   - 범위를 벗어난 좌표 처리

클래스명: CoordinateDisplay

📝 STEP 6: 하이브리드 위치 추적 시스템
================================================================================

Unity C# 스크립트를 작성해주세요.

목표: 스캔된 영역과 미스캔 영역을 구분하여 사용자 위치 추적

요구사항:
1. HybridLocationTracker 클래스
   - 사용자의 현재 위치 추적
   - 어느 지역에 있는지 판단
   - 정확도 레벨 결정

2. LocationAccuracy Enum:
   - None: 위치 추적 불가
   - Estimated: 추정 위치 (±1m 이상)
   - Accurate: 정확한 위치 (±10cm)

3. 핵심 함수들:
   - IsInScannedRegion(Vector2): 스캔 영역 확인
   - GetUserPosition(): 사용자 위치 반환
   - DetermineAccuracy(): 정확도 판단
   - GetMarkerColor(): 정확도에 따른 마커 색상 반환

4. 지역 정의:
   - 스캔된 영역 (전시실): (5,5) ~ (35,25)
   - 미스캔 영역 (로비): (0,0) ~ (5,40)
   - 미스캔 영역 (복도): (35,0) ~ (50,40)

5. 시뮬레이션:
   - 3가지 위치에서 테스트
   - 각 위치에서의 정확도 출력
   - 마커 색상 변경

6. 출력 형식:
   - "위치: X, Z"
   - "영역: 전시실"
   - "정확도: 정확함 (±10cm)"
   - "마커색: Red"

클래스명: HybridLocationTracker

📝 STEP 7: 통합 시스템 (선택사항 - 모든 기능 연동)
================================================================================

Unity C# 스크립트를 작성해주세요.

목표: 모든 시스템을 통합하여 완전한 AR 네비게이션 시뮬레이션

요구사항:
1. ARNavigationSystem 클래스
   - 모든 이전 시스템 통합
   - 전체 흐름 관리

2. 주요 기능:
   - Immersal 위치 가져오기
   - 통합 좌표 변환
   - 2D 지도 좌표로 변환
   - 정확도 판단
   - 크로스맵 네비게이션 표시

3. 시뮬레이션:
   - 10프레임 실행
   - 각 프레임에서 사용자 위치 업데이트
   - 맵1 → 맵2 전환 시뮬레이션
   - 전체 상태 출력

4. 출력 형식:
   [프레임 1]
   ├─ Immersal 위치: (3, 0, 7)
   ├─ 통합 좌표: (3, 0, 7)
   ├─ 2D 지도: (3, 7)
   ├─ Canvas: (X, Y)
   ├─ 영역: 전시실
   └─ 정확도: 정확함 ✅

5. 에러 처리:
   - 유효하지 않은 위치 처리
   - 범위 벗어남 처리

클래스명: ARNavigationSystem
