====================================================
  듀얼 맵 캘리브레이션 가이드
  맵 A: 145962 (AR Space)  /  맵 B: 145963 (AR Space 2)
====================================================

[ 개요 ]
두 개의 Immersal 맵(145962, 145963)은 각자의 로컬 좌표계를 가집니다.
겹치는 구간(overlap)에서 두 좌표계의 상대 오프셋을 구해
AR Space 2의 Transform에 적용하는 것이 캘리브레이션의 목표입니다.


====================================================
  방법 A: PLY 파일 기반 사전 계산
====================================================

필요 도구: CloudCompare (무료, https://cloudcompare.org)

절차:
  1. CloudCompare 실행
  2. File → Open → Assets/Maps/145962-real-sparse.ply 열기
  3. File → Open → Assets/Maps/145963-real2-sparse.ply 추가 열기
  4. 두 포인트클라우드에서 동일한 물리적 특징점 3개 이상 식별
       예: 벽 모서리, 기둥 끝, 바닥 패턴 교차점 등
  5. Edit → Manual Segmentation 으로 공통 영역 선택
  6. Registration → Fine Registration (ICP) 실행
       → 변환 행렬 (4×4 Matrix) 출력됨
  7. 행렬에서 값 추출:
       - translation: m03(x), m13(y), m23(z)
       - rotation: 행렬 좌상단 3×3 → 오일러각 또는 쿼터니언으로 변환
  8. Unity Editor에서 AR Space 2 선택
       → Transform Position에 translation 값 입력
       → Transform Rotation에 rotation 값 입력
  9. 씬 저장 (Ctrl+S)

TIP: ICP 정합 오차(RMS)가 0.05m 이하이면 충분히 정밀합니다.


====================================================
  캘리브레이션 후 검증
====================================================

  1. 맵 A(145962) 구역의 웨이포인트 → 실제 위치와 일치하는지 확인
  2. 맵 B(145963) 구역의 웨이포인트 → 실제 위치와 일치하는지 확인
  3. 두 맵 경계(overlap 구간)에서 유도선이 자연스럽게 연결되는지 확인
  4. 화살표 방향이 경계 통과 시 튀지 않는지 확인

  웨이포인트가 실제 위치와 어긋날 경우:
    → AR Space 2의 Position/Rotation을 소폭 조정 (0.01~0.05m 단위)
    → 조정 후 씬 저장 및 재빌드
